# 范长俊 学术主页 — 使用与部署指南

本站基于 [academicpages](https://github.com/academicpages/academicpages.github.io)（Jekyll 模板）定制，已加入**中英文一键切换**功能，并填入了你的个人信息与 Google Scholar 论文。日常维护只需修改对应的 Markdown / YAML 文件即可。

> 重要：真正要上传到 GitHub 的，是**含有 `_config.yml` 的这个文件夹**（即你现在看到这份指南所在的目录）。不要把外层多套了一层的 `academicpages.github.io-master` 一起传上去。

---

## 一、上线前你需要补充的内容（5 分钟）

1. **头像**：把你的照片命名为 `profile.jpg` 放到 `images/` 目录，然后打开 `_config.yml`，把
   `avatar : "profile.svg"` 改成 `avatar : "profile.jpg"`。（不改也能用，会显示一张图网络风格的占位头像）
2. **邮箱**：打开 `_config.yml`，找到 `email :`，填上你的邮箱（如 `xxx@nudt.edu.cn`）。
3. **简历 PDF（可选）**：把简历命名为 `cv.pdf` 放到 `files/` 目录，CV 页里的「下载 PDF」链接即可生效。
4. **教育背景**：打开 `_pages/cv.md`，把标注「请在此补充」的学士/硕士经历填上。

> 这些都不是必须项，留空也能正常上线，随时可改。

## 二、部署到 GitHub Pages

你的网址将是：**https://fanchangjun.github.io**

### 方法 A：用 GitHub Desktop（最简单，推荐，不用敲命令）

1. 在 GitHub 上新建一个仓库，名字**必须**是 `fanchangjun.github.io`（设为 Public 公开）。
2. 下载安装 [GitHub Desktop](https://desktop.github.com/)，并登录你的账号。
3. 菜单 `File → Add local repository`，选择**含 `_config.yml` 的这个文件夹**。
4. 如果提示「这不是一个 Git 仓库，是否创建」，点 `create a repository`。
5. 在左下角填写一句提交说明（如 `init my homepage`），点 `Commit to main`。
6. 点右上角 `Publish repository`，仓库名选 `fanchangjun.github.io`，去掉「Keep this code private」勾选，发布。
7. 打开仓库网页 → `Settings → Pages`，确认 Source 是 `Deploy from a branch`，分支选 `main`、目录选 `/ (root)`，保存。
8. 等 1–3 分钟，访问 **https://fanchangjun.github.io** 即可。

### 方法 B：用命令行 Git

在含 `_config.yml` 的目录下打开终端，依次执行：

```bash
git init
git add .
git commit -m "init my homepage"
git branch -M main
git remote add origin https://github.com/fanchangjun/fanchangjun.github.io.git
git push -u origin main
```

然后到仓库 `Settings → Pages` 确认分支为 `main` / 目录 `/ (root)`，等待构建完成。

### 方法 C：网页直接上传

在 GitHub 新建 `fanchangjun.github.io` 仓库后，点 `Add file → Upload files`，把本文件夹内**所有文件和文件夹**拖进去提交。文件较多时可分几次拖。完成后同样到 `Settings → Pages` 开启。

## 三、日常维护（都是改 Markdown）

| 想改什么 | 改哪个文件 |
| --- | --- |
| 个人简介 / 研究方向 / 近期动态 | `_pages/about.md` |
| 个人简历 | `_pages/cv.md` |
| 新增一篇论文 | 在 `_publications/` 里新建一个 `年-月-日-标题.md` |
| 新增一场学术报告 | 在 `_talks/` 里新建 `年-月-日-标题.md`（已有示例可直接复制） |
| 新增一门课程 | 在 `_teaching/` 里新建 `年-月-日-标题.md`（已有示例可直接复制） |
| 姓名 / 头衔 / 邮箱 / 各类链接 | `_config.yml` 的 `author:` 部分 |
| 顶部导航栏 | `_data/navigation.yml` |

每次改完，重复一次「提交 + 推送」（GitHub Desktop 里就是 Commit 再 Push），网站会在几分钟内自动更新。

## 四、关于中英文切换（重要）

- 切换按钮在网页**右上角的 `EN` / `中` 胶囊按钮**，点一下即可整站切换；选择会被记住（下次访问保持上次的语言）。默认中文。
- 凡是需要双语的地方，都写成**两段**：
  ```html
  <div class="lang-zh" markdown="1">
  这里写中文……
  </div>

  <div class="lang-en" markdown="1">
  English here...
  </div>
  ```
  行内短语用 `<span class="lang-zh">中文</span><span class="lang-en">English</span>`。
- 系统会根据当前语言**自动只显示对应一种**。所以新增内容时，记得中、英各写一份；只写中文也可以，英文模式下该段就会留空。
- 论文标题等本身是英文，无需翻译，中英两种模式都按英文原文显示。

## 五、本地预览（可选，需要 Ruby 环境）

不预览、直接推到 GitHub 让其自动构建也完全没问题。如果想本地看效果：

```bash
gem install bundler
bundle install
bundle exec jekyll serve -l -H localhost
```

然后浏览器打开 `http://localhost:4000`。
